## 💡 面试话术模板

### 项目介绍（1分钟）

"我开发了一个面向IoT设备的自动化测试管理平台，主要解决智能硬件测试中的管理和自动化问题。

**技术架构：**基于Spring Cloud Alibaba微服务架构，拆分了网关、认证、项目管理、测试用例等4个核心服务。使用Nacos实现服务注册和配置管理，通过Gateway进行统一路由和JWT认证。

**核心功能：**支持IoT设备管理（通过MQTT协议通信）、测试用例管理、自动化测试执行、测试报告生成等完整流程。

**业务价值：**这个项目结合了我在Anker的扫地机测试实习经验，针对智能硬件的测试场景设计，支持冒烟测试、专项测试等多种测试类型。

**技术亮点：**
1. 集成MQTT协议实现与设备的实时通信和指令下发
2. 使用JWT实现无状态认证，支持分布式部署
3. 通过Docker Compose实现一键部署所有基础设施"

### 技术深度问题

**Q1：为什么选择微服务架构？**

答：基于以下考虑：
1. 扩展性：测试执行模块负载较高，可以单独扩容
2. 技术栈隔离：认证模块使用Spring Security，测试模块使用MQTT，职责清晰
3. 团队协作：不同团队可以独立开发各自的服务
4. 故障隔离：某个服务挂掉不影响其他服务

**Q2：如何实现设备与平台的通信？**

答：基于MQTT协议：
1. 设备订阅`device/command/{deviceId}`主题接收测试指令
2. 设备发布到`device/status/{deviceId}`主题上报状态和日志
3. 后端通过EMQX作为消息代理，实现订阅和发布
4. 使用WebSocket推送实时日志到前端

**Q3：如何防止并发执行同一测试用例？**

答：使用Redis分布式锁：
```java
String lockKey = "testcase:lock:" + caseId;
if (redisTemplate.opsForValue().setIfAbsent(lockKey, "1", 5, TimeUnit.MINUTES)) {
    try {
        // 执行测试
    } finally {
        redisTemplate.delete(lockKey);
    }
} else {
    throw new BusinessException("该用例正在执行中");
}
```
